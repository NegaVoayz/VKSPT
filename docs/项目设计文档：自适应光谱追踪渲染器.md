# 项目设计文档：自适应光谱追踪渲染器

## 一、项目概述

### 1.1 项目定位
基于 Vulkan 1.4 + Ray Query 实现的**物理正确的色散渲染器**，采用自适应光谱分裂策略，利用 RTX 4060 及以上硬件的 RT Core 加速，实现实时/交互级性能。语言标准C++20，CMake版本4.3.1

### 1.2 核心特性
- 连续光谱建模（380nm-780nm）
- 自适应波长区间分裂
- 色散向量传播（支持多次反射/折射）
- GPU 光线重排序优化
- 硬件加速求交（RT Core）

## 三、核心数据结构

### 3.1 SpectralRay

```cpp
struct SpectralRay {
    // === 几何信息 ===
    vec3 origin;           // 当前位置
    vec3 direction;        // 传播方向（归一化）
    
    // === 色散信息 ===
    vec3 dispersion;       // 色散向量 = 单位距离内紫边相对于中心的横向偏移
                           // 垂直于 direction，长度 = |dir_violet - dir_center|
    
    // === 光谱信息 ===
    int lambda_start;      // 波长区间起点 (nm, 380-780, 整数)
    int lambda_end;        // 波长区间终点 (闭区间)
    
    // === 物理量 ===
    vec3 energy;           // RGB 能量权重（被材质吸收后衰减）
    vec3 last_split;       // 上次分裂/表面交互时记录的横向分离向量
                           // 总横向分离 = last_split + dispersion * distance
    
    // === 追踪控制 ===
    int bounce;            // 剩余弹射次数
    int generation;        // 分裂代数（限制最大深度）
};
```

### 3.2 Material

```cpp
struct Material {
    int type;              // DIELECTRIC, METAL, LAMBERTIAN
    
    // 色散参数（柯西公式: n = A + B/λ²）
    vec3 cauchy_A;         // A 系数 (RGB)
    vec3 cauchy_B;         // B 系数 (RGB)
    
    // 光学属性
    vec3 albedo;           // 基础颜色/反射率
    vec3 absorption;       // 吸收系数（比尔-朗伯定律）
    float roughness;       // 粗糙度（用于模糊反射）
    float ior;             // 基准折射率（用于无色散近似）
};
```

### 3.3 RayAction

```cpp
enum RayAction {
    TERMINATE,   // 能量耗尽或超出弹射次数
    SPLIT,       // 需要分裂
    REFRACT,     // 折射
    REFLECT,     // 反射
    MISS         // 未命中，累加背景色
};
```

### 4.2 分裂决策

```glsl
bool shouldSplit(SpectralRay ray, float distance_to_next_event) {
    // 预测到达下一个事件时的总横向分离
    vec3 predicted_sep = ray.last_split + ray.dispersion * distance_to_next_event;
    float predicted_magnitude = length(predicted_sep);
    
    // 条件1：预测分离超过像素阈值
    float pixel_threshold = getPixelSizeAtDistance(distance_to_next_event);
    if (predicted_magnitude > pixel_threshold) return true;
    
    // 条件2：波长区间足够宽（避免无限分裂）
    if (ray.lambda_end - ray.lambda_start < 10) return false;
    
    // 条件3：能量足够（低能量不必分裂）
    if (length(ray.energy) < 0.05) return false;
    
    // 条件4：分裂代数限制
    if (ray.generation > 5) return false;
    
    return false;
}
```

### 4.3 分裂执行

```glsl
void splitRay(SpectralRay ray, out SpectralRay left, out SpectralRay right) {
    int lambda_mid = (ray.lambda_start + ray.lambda_end) / 2;
    float total_width = ray.lambda_end - ray.lambda_start;
    float left_width = lambda_mid - ray.lambda_start;
    float right_width = ray.lambda_end - lambda_mid;
    
    // 左子光线（紫端）
    left = ray;
    left.lambda_end = lambda_mid;
    left.energy = ray.energy * (left_width / total_width);
    left.generation = ray.generation + 1;
    // last_split 继承，不改变
    
    // 右子光线（红端）
    right = ray;
    right.lambda_start = lambda_mid;
    right.energy = ray.energy * (right_width / total_width);
    right.generation = ray.generation + 1;
}
```

### 4.4 色散向量计算（折射时）

```glsl
vec3 computeDispersion(vec3 incident, vec3 normal, 
                        float lambda_center, float lambda_violet,
                        Material mat) {
    // 中心波长方向
    vec3 dir_center = refract(incident, normal, 1.0, ior(lambda_center, mat));
    // 紫边波长方向
    vec3 dir_violet = refract(incident, normal, 1.0, ior(lambda_violet, mat));
    
    // 色散 = 紫边方向 - 中心方向
    vec3 dispersion = dir_violet - dir_center;
    // 移除沿传播方向的分量（色散是横向的）
    dispersion = dispersion - dot(dispersion, dir_center) * dir_center;
    
    return dispersion;
}
```

### 4.5 last_split 变换规则

| 事件               | 变换规则                                                     | 说明           |
| ------------------ | ------------------------------------------------------------ | -------------- |
| **传播**           | `last_split += dispersion * distance`                        | 累积横向分离   |
| **折射**           | `last_split = rotate_to_new_frame(last_split, old_dir, new_dir)` | 变换到新切平面 |
| **反射**           | `last_split = reflect(last_split, normal)`                   | 镜像反射       |
| **分裂**           | `last_split` 不变                                            | 子光线继承     |
| **进入无色散介质** | `last_split` ，`dispersion` 不变                             | 不再增加分离   |

### 4.6 二阶勒让德衰减展开

#### 动机

传统路径追踪用标量 throughput 表示路径衰减（Beer-Lambert 吸收 × Fresnel 反射损失等）。但 Cauchy 体积吸收 α(λ) = A + B/λ² 是波长相关的——不同波长的光穿过同一介质后衰减不同。当前系统通过**力分裂**（force-split）将宽带 beam 分成许多窄带 beam，每条窄带用其中心波长的吸收系数近似——代价是 beam 数量指数增长。

**核心改进**：将衰减函数 D(λ) = exp(−τ(λ)) 在 beam 的波长范围上做二阶勒让德展开，用 3 个系数 (c₀, c₁, c₂) 替代单一标量 throughput。一条宽带 beam 即可准确表达波长相关的衰减变化，大幅减少吸收驱动的分裂。

#### 勒让德多项式 ([-1, 1])

基函数：
- P₀(x) = 1
- P₁(x) = x
- P₂(x) = (3x² − 1) / 2

正交性：∫₋₁¹ P_m(x) P_n(x) dx = 2/(2n+1) × δ_{mn}

#### 波长归一化

对波长范围 [λ_start, λ_end] 的 beam，映射到归一化坐标 x ∈ [−1, 1]：

```
λ_mid = (λ_start + λ_end) / 2        （中心波长，nm）
Δ     = (λ_end − λ_start) / 2        （半宽，nm）
x(λ)  = (λ − λ_mid) / Δ              （归一化坐标）
λ(x)  = λ_mid + Δ·x                   （逆映射）
```

#### 衰减函数

每条 beam 在其波长范围内有衰减函数 D(x) = exp(−τ(x))，其中 τ(x) 是沿路径累积的光学深度（所有 Cauchy 吸收之和）：

τ(x) = Σ_i (A_i + B_i / [λ(x)/1000]²) × t_i  （λ 以 nm 计，需转换为 μm）

二阶勒让德展开：

```
D(x) ≈ c₀·P₀(x) + c₁·P₁(x) + c₂·P₂(x)
     = c₀ + c₁·x + c₂·(3x²−1)/2
```

系数（正交投影）：c_n = ((2n+1)/2) × ∫₋₁¹ D(x) P_n(x) dx

#### 3 点 Gauss-Legendre 求积

在 [−1, 1] 上的节点和权重：

| k | x_k | w_k |
|---|-----|-----|
| 0 | −√(3/5) ≈ −0.77459667 | 5/9 ≈ 0.55555556 |
| 1 | 0                    | 8/9 ≈ 0.88888889 |
| 2 | +√(3/5) ≈ +0.77459667 | 5/9 ≈ 0.55555556 |

3 点求积对 ≤5 次多项式精确。

#### 吸收更新

穿过介质时，旧衰减 D_old(x) 乘以新表面的 Beer-Lambert 衰减：

对每个求积节点 x_k：
```
λ_k     = λ_mid + Δ·x_k                (nm)
λ_k_μm  = λ_k / 1000                    (μm)
α_k     = A + B / λ_k_μm²              (Cauchy 吸收系数)
D_old,k = c₀ + c₁·x_k + c₂·(3x_k²−1)/2
D_new,k = D_old,k × exp(−α_k × tHit)
```

投影回勒让德系数：
```
c₀_new = ½ × Σ_k w_k × D_new,k
c₁_new = ³⁄₂ × Σ_k w_k × D_new,k × x_k
c₂_new = ⁵⁄₂ × Σ_k w_k × D_new,k × P₂(x_k)
```

其中 P₂(x₀) = P₂(x₂) = 0.4，P₂(0) = −0.5。

#### 拟合误差检测

二阶勒让德展开是对指数函数的近似——当吸收变化剧烈时拟合可能不足。在检查点 x = 0.5 处比较精确衰减与勒让德近似：

```
D_exact(0.5) = D_old(0.5) × exp(−α(λ_cp) × tHit)          （精确）
D_fit(0.5)   = c₀_new + 0.5·c₁_new − 0.125·c₂_new        （拟合）
相对误差     = |D_exact − D_fit| / max(c₀_new, 0.001)
```

若相对误差 > 2%，说明二阶不足以拟合，触发吸收驱动的分裂。

#### 分裂重投影

父 beam 有系数 (c₀,c₁,c₂) 定义在 [λ_pStart, λ_pEnd] 上。分裂后子 beam 的波长范围是 [λ_cStart, λ_cEnd]。

对子 beam 的每个求积节点 x_k，映射回父 beam 的归一化坐标：
```
λ_k       = λ_cMid + Δ_c·x_k
x_parent  = (λ_k − λ_pMid) / Δ_p      （clamp 到 [−1,1]）
D(λ_k)    = c₀_parent + c₁_parent·x_parent + c₂_parent·P₂(x_parent)
```

然后用相同的求积投影公式计算子 beam 的新勒让德系数。

#### 天空贡献（最终累积）

宽带 beam（λ_range > 20nm）：勒让德展开的平均值即 c₀（因 ∫₋₁¹ P₁ = ∫₋₁¹ P₂ = 0）：
```
contribution = skyColor × c₀ × wlFrac
```

窄带 beam（λ_range ≤ 20nm）：在中心波长处求值（x = 0）：
```
D(0)  = c₀ − c₂/2     （因为 P₁(0)=0, P₂(0)=−½）
contribution = skyColor × wavelengthToRGB(λ_mid) × D(0) × wlFrac
```

### 4.7 CIE 1931 三刺激值与 Wiener 光谱反演

#### 动机

替换 §4.6 中简化的天空贡献公式。原有 `wavelengthToRGB()` 是手工分段函数，物理不正确；环境贴图仅有 RGB，无光谱信息。

#### 显示参考 CMF

将 CIE 1931 2° 标准观察者色匹配函数 (x̄,ȳ,z̄) 预转换为显示参考的线性 sRGB 响应：
```
[CMF_R(λ), CMF_G(λ), CMF_B(λ)]^T = XYZ_TO_SRGB × [x̄(λ), ȳ(λ), z̄(λ)]^T
```
然后逐通道归一化使等能光谱 S(λ)≡1 产生线性 sRGB (1,1,1)。

81 个采样点（380–780nm, Δλ=5nm），线性插值。

#### Wiener 伪逆法 RGB→光谱

系统矩阵 **M** (3×81) 将光谱 **s** 映射到线性 sRGB：
```
c = M · s,   M[i,j] = CMF_i(λ_j) × Δλ
```

伪逆：**M^+** = **M**^T (**M M**^T)^(−1)  (81×3)，其列向量为光谱基函数：
- `BASIS_R(λ)`：产生线性 sRGB = (1,0,0) 的最小范数光谱
- `BASIS_G(λ)`：产生线性 sRGB = (0,1,0) 的最小范数光谱
- `BASIS_B(λ)`：产生线性 sRGB = (0,0,1) 的最小范数光谱

任意光源的光谱估计：
```
Ŝ(λ) = lin_R × BASIS_R(λ) + lin_G × BASIS_G(λ) + lin_B × BASIS_B(λ)
```

#### Gauss-Legendre 天空积分

光束命中天空时，对光谱×衰减×CMF 三重积做 3 点求积（复用 §4.6 的节点和权重）：
```
对每个求积节点 x_k (k=0,1,2):
  λ_k      = λ_mid + halfWidth × x_k
  S_k      = Ŝ(λ_k)                          (Wiener 估计)
  D_k      = evalDecay(c₀,c₁,c₂, x_k)        (勒让德衰减)
  cmf_k    = [CMF_R(λ_k), CMF_G(λ_k), CMF_B(λ_k)]
  d(sRGB) += w_k × S_k × D_k × cmf_k × halfWidth
```
结果经 gamma 校正后直接累加到像素。

消除窄带/宽带分支——求积自动在所有带宽上精确。

## 五、优化策略

### 5.1 光线重排序（Sorting）

**问题**：不同光线的分裂决策不同，导致线程束发散，GPU 利用率下降。

**方案**：每轮迭代后，将所有光线按 `RayAction` 分类到不同桶。

```cpp
// 阶段1：分类
// 在 Compute Shader 中遍历所有活跃光线
RayAction action = classifyRay(ray);
uint bucket_index = getBucketIndex(action);  // 0-4
uint offset = atomicAdd(counter[bucket_index], 1);
output_indices[offset] = ray_index;

// 阶段2：重排
// 使用前缀和确定每个桶的起始位置
uint prefix[5];
prefix[0] = 0;
for (int i = 1; i < 5; i++) {
    prefix[i] = prefix[i-1] + counter[i-1];
}

// 阶段3：分桶执行
// 为每个桶单独 dispatch，每个桶内光线执行相同代码路径
dispatch(SPLIT_BUCKET, counter[SPLIT_BUCKET]);
dispatch(REFRACT_BUCKET, counter[REFRACT_BUCKET]);
// ...
```

### 5.2 其他优化

| 优化               | 方法                                    | 预期收益        |
| ------------------ | --------------------------------------- | --------------- |
| **预计算折射率表** | 波长 → n 的 LUT（纹理），避免运行时计算 | 30-50% 减少指令 |
| **自适应阈值**     | 按屏幕空间像素大小动态调整分离阈值      | 远处物体少分裂  |
| **能量截断**       | `length(energy) < 0.01` 的光线直接终止  | 减少无效光线    |
| **区间合并**       | 方向相近、波长相邻的光线可合并          | 降低光线总数    |
| **双缓冲队列**     | ping-pong 缓冲区存储每轮光线            | 避免重新分配    |
| **复用 Ray Query** | 每个线程复用一个 query 对象             | 减少寄存器压力  |

| 复杂（玻璃、水、钻石）   | 1080p  | 4-16          | 15-30 FPS |

## 八、关键风险与对策

| 风险               | 影响                   | 对策                                   |
| ------------------ | ---------------------- | -------------------------------------- |
| 光线数量爆炸       | 内存溢出/性能崩溃      | 限制分裂代数(≤5) + 能量截断 + 区间合并 |
| 线程发散严重       | GPU 利用率低           | 重排序 + 按 Action 分桶 dispatch       |
| Ray Query 过多激活 | 占用寄存器，降低并行度 | 每个线程复用同一个 query 对象          |
| 调试困难           | 难以定位着色器 bug     | CPU 参考实现验证核心算法               |
| 数值稳定性         | 色散向量累积误差       | 定期归一化，使用 double 中间计算       |

### B. 波长转 RGB 快速函数

```glsl
vec3 wavelengthToRGB(float lambda) {
    vec3 rgb = vec3(0.0);
         if (lambda < 440.0) rgb = vec3(0.5 - 0.5 * (lambda - 380.0) / 60.0, 0.0, 1.0);
    else if (lambda < 490.0) rgb = vec3(0.0, (lambda - 440.0) / 50.0, 1.0);
    else if (lambda < 510.0) rgb = vec3(0.0, 1.0, 1.0 - (lambda - 490.0) / 20.0);
    else if (lambda < 580.0) rgb = vec3((lambda - 510.0) / 70.0, 1.0, 0.0);
    else if (lambda < 645.0) rgb = vec3(1.0, 1.0 - (lambda - 580.0) / 65.0, 0.0);
    else if (lambda <= 780.0) rgb = vec3(1.0, 0.0, 0.0);
    
    float attenuation = 1.0;
    if (lambda < 420.0) attenuation = 0.3 + 0.7 * (lambda - 380.0) / 40.0;
    if (lambda > 700.0) attenuation = 0.3 + 0.7 * (780.0 - lambda) / 80.0;
    return rgb * attenuation;
}
```
