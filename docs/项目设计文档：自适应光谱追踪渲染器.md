# 项目设计文档：自适应光谱追踪渲染器

## 一、项目概述

### 1.1 项目定位
基于 Vulkan 1.4 Ray Tracing Pipeline 实现的光谱色散渲染器，采用 Slang→SPIR-V 着色器、光子映射间接光照、渐进式跨帧累积。C++23 + `vk::raii`，CMake 构建。

### 1.2 核心特性
- 连续光谱建模（380–780nm, 10×40nm桶）
- Legendre对数衰减展开（3系数替代标量throughput）
- 光谱色散向量传播（D/V: 偏移+增长率, 多次反射/折射）
- 自适应光谱分裂（几何驱动, 2^k路子射线, Bresenham波长分区）
- 光子映射间接光照（空间哈希 + I/S分离光谱平滑）
- 渐进式跨帧累积 + 2-scale边缘保持降噪

## 三、渲染架构

### 3.1 整体流程

```
每帧:
  Renderer::RenderFrame()
    ├─ 获取交换链图像
    ├─ 跨帧累积检测(相机移动→重置)
    ├─ bindFrameDescriptors() — 20个descriptor绑定
    ├─ FrameRecorder::record()
    │   ├─ transitionImages() — Undefined→General / shader-write barrier
    │   ├─ fillBuffer清零 rayStats, displayCellData/gatheredCellData(首帧)
    │   ├─ dispatchBlendPhoton() — EMA blend gatheredCellData→displayCellData
    │   ├─ barrier displayCellData (compute → RT)
    │   ├─ dispatchTrace() — vkCmdTraceRaysKHR(camera raygen, 1280×720×1)
    │   │   └─ shader: RayGenMain → TraceRay → ClosestHitMain/MissMain
    │   ├─ denoisePass() — 2-scale边缘保持降噪(compute, 8×8工作组)
    │   ├─ dispatchStatsOverlay() — F3切换(compute, 35×23×1)
    │   └─ copyOutputToSwapchain() — 输出→交换链+Present布局转换
    └─ FrameRecorder::submit() — 提交+呈现+GPU时间戳回读

异步(独立command buffer, 与渲染并行):
  PhotonRecorder::trySubmitPhotonBatch()
    └─ recordPhotonBatch() → 光子追踪 + 5级hash管线(见§4)
```

### 3.2 SBT (Shader Binding Table)

4个shader group:
| 索引 | 类型 | 用途 |
|------|------|------|
| 0 | raygen | 相机路径 RayGenMain |
| 1 | closest hit | 材质处理 ClosestHitMain |
| 2 | miss | 天空/环境 MissMain |
| 3 | raygen | 光子路径 PhotonRayGenMain |

Camera dispatch用region[0,1,2], Photon dispatch用region[3,1,2].

### 3.3 Descriptor布局

所有shader共享单一descriptor set layout (set=0, 20个binding):

| 槽 | 类型 | 内容 |
|----|------|------|
| 0 | AS | TLAS |
| 1 | StorageImage | output (RGBA8) |
| 2 | UBO | materials[16] (4×float4/std140) |
| 3 | UBO | lights[8] |
| 4–6 | SSBO | vertex/index/instanceRanges |
| 7 | SSBO | photonBuffer (MAX_PHOTONS × 88B) |
| 8 | SSBO | photonCounter |
| 9 | SSBO | hashCellData (HASH_TABLE_SIZE × 2 uint) |
| 10 | SSBO | sortedPhotonIndices |
| 11 | CombinedImageSampler | envMap |
| 12 | SSBO | normalData (smooth normals) |
| 13 | SSBO | accumData (RGBA per pixel) |
| 14 | StorageImage | normalImage (RGBA16f G-buffer) |
| 15 | StorageImage | depthImage (R32f G-buffer) |
| 16 | SSBO | instanceNormals (3×4 normal matrices) |
| 17 | SSBO | rayStats |
| 18 | SSBO | cellPhotonData (23 floats/cell) |
| 19 | SSBO | gatheredCellData (13 uints/cell) |

3个descriptor set: 2个用于双缓冲相机帧, 1个用于异步光子管线(PHOTON_SET).

Push constant: 256字节, 匹配 `PushConstants` / `TracePushConstant`.

## 四、光子映射系统

### 4.1 异步渐进式构建

每帧从renderer调用 `trySubmitPhotonBatch()`, 提交一个光源批次的光子追踪+hash管线, 在独立command buffer上执行, 与相机渲染并行.

```
setupPhotons() — 首帧: 统计活跃光源, 创建command pool/fence, 绑定PHOTON_SET
trySubmitPhotonBatch() — 每帧: 如前一批fence已signal且有剩余光源
  ├─ 清零 photonCounter + hashCellData
  ├─ dispatchPhotonTraceBatch() — vkCmdTraceRaysKHR(photon raygen)
  │   └─ PhotonRayGenMain: 光源采样 → TraceRay
  │       ClosestHitMain(PASS_PHOTON) → handlePhotonHit()
  │         ├─ 漫反射: storePhoton() → photonBuffer (10桶光谱×Legendre衰减)
  │         └─ 镜面: handleDielectric()/handleMetal() 递归弹射
  └─ recordHashStages() — 5个compute dispatch
```

### 4.2 5级Hash管线

| 阶段 | 功能 | 工作组 |
|------|------|--------|
| `hash_count` | 每个光子计算空间cell hash, 原子递增计数器 | 256, photonCount×16/256 |
| `hash_scan` | 单工作组前缀和 → scatter偏移, 清零计数 | 1, 1024 threads |
| `hash_scatter` | 光子索引按偏移写入排序数组 | 256 |
| `hash_aggregate` | 每cell聚合: 位置/法线/方向均值 + 光谱求和(23 floats/cell) | 256, 1024 groups |
| `hash_gather` | I/S分离: 目标辐照度(I_t)×邻域平滑光谱(Ŝ), CAS-loop原子累加 | 256, 1024 groups |

### 4.3 空间哈希

- HASH_TABLE_SIZE = 262144 cells, cellSize = 0.02m
- 哈希函数: Morton 3D交错 + 高位雪崩混合 → `computeCellHash(int3)`
- `cellPhotonData`: 23 floats/cell (pos+nrm+dir各3 + 10光谱 + cnt)
- `gatheredCellData`: 13 uints/cell (10光谱asfloat + 3位置作碰撞检测)

### 4.4 I/S分离光谱平滑 (hash_gather)

```
I_t   = Σ_b specSum_t[b]                    — 目标cell辐照度(保持锐利)
Φ_j   = 邻域cell总光谱能量                    — 权重
G(d)  = exp(-d²/(2σ²))                      — 空间高斯, σ = maxGatherRadius/2
num[b] = Σ_j G(d) × specSum_j[b]             — 邻域加权光谱
denom  = Σ_j Φ_j × G(d)                      — 能量加权归一化
output[b] = I_t × num[b] / (denom × 0.5π·cellSz²)
```

防护: 邻域光子密度检测(minNeighborPhotons), 孤立cell剔除(邻域能量<自身50%), 离群值压制(I_t > 2×邻域均值→clamp).

### 4.5 相机路径光子收集

`gatherPhotons()` — 曼哈顿7邻域(自cell+6面相邻), Gaussian权重: center=1.0, face=exp(-1/(2σ²))≈0.607 (σ=1.0). Hash碰撞检测(存储位置验证). 光谱累加后归一化到 `p.spectral[10]`.

### 4.6 时间EMA降噪 (blend_photon)

每帧在trace之前运行: `displayCellData = α·gatheredCellData + (1-α)·displayCellData`, α=0.01.
渐进式平滑光子图, 配合hash_gather纯cell输出(无空间混合). 低α值保证时域稳定性.

## 五、光谱系统

### 5.1 数据结构

**Payload** (每射线状态, `shaders/rt/payload.slang`):
```
color, hitT, normal
lamStart, lamEnd           — 波长范围(nm整数)
D, V                       — 色散偏移向量, 单位距离增长率(均⟂rayDir)
throughput, decayC1, decayC2 — Legendre log-decay系数
bounce, generation         — 弹射/分裂代数
spectral[10]               — 40nm桶光谱累积
rng, specularChain, lightRGB, isShadowed
```

**PushConstants** (256字节, `FrameRecorder.h::TracePushConstant`):
相机参数, SPP, maxBounces, 材质/光源数量, split阈值, 光子图参数, batch信息, FPS.

### 5.2 Legendre对数衰减 (§4.6)

传统标量throughput → 2阶Legendre展开 `(c₀, c₁, c₂)` 在log空间表示波长相关衰减:
```
D(x) ≈ exp(c₀ + c₁·P₁(x) + c₂·P₂(x)),  x ∈ [-1,1] 归一化波长
```

操作使用3点Gauss-Legendre求积 (节点±0.7746, 0; 权重5/9, 8/9, 5/9):
- 吸收: 每节点计算Cauchy α(λ), 乘Beer-Lambert, 投影回Legendre
- 菲涅尔: 每节点eval反射率, log乘, 投影回Legendre
- 分裂重投影: 子beam波长范围→父beam归一化坐标, 求积投影

### 5.3 色散传播

```
ClosestHitMain:
  1. 计算交点信息(位置/法线/材质)
  2. RR+弹射限制 → 死亡检测
  3. Split检查: 纯几何, 2·|D⟂| > forceSplitWidth → 分裂, 父射线死亡
     traceSpectralSplit(): 子射线从入射方向重击表面, 独立走完整材质处理
  4. apply_diff: D += V·hitT, 投影⟂rayDir  
  5. 材质dispatch
```

材质处理中D/V变换:
- 反射: `D' = D - 2·(D·n)·n`, V同理
- 折射: D/V旋转到新方向 + `V += computeDispersion()` (Cauchy IOR差异)

天空命中色散补偿: `Vperp *= params.splitMult` (默认0.25). 补偿玻璃平行面entry+exit色散抵消(理想平行面净色散为0), 同时缩小per-bucket天空采样角度偏移, 提高快速路径命中率.

### 5.4 光谱分裂

`traceSpectralSplit()`:
```
k = ceil(log2(2|D⟂| / forceSplitWidth)), n = 2^k
s = 2^(-k), childD = s·D, childV = s·V
对i=0..n-1:
  Sj = -(1-s) + i·2s + jitter   — S参数(位移比例)
  childDir = normalize(baseDir + Sj·V)
  childC2 = s²·C2
  childC1 = s·(C1 + 3·Sj·C2)
  childC0 = C0 + Sj·C1 + (1.5·Sj²-0.5+0.5·s²)·C2
  lamRight = Bresenham整数分区 + jitter
```

### 5.5 RGB ↔ 光谱转换

**数据基础** (`cmf_data.slang`):
- CIE 1931 2° CMF → sRGB线性矩阵 → 81采样点(380-780nm, Δ5nm)预积分
- 10个40nm桶的预积分CMF: `BUCKET_CMF[10]` (各桶内CMF积分, 直接到线性sRGB)
- Wiener基函数: `BASIS_R/G/B[81]` — 由sRGB→XYZ→CMF伪逆计算, 用于RGB→光谱重建

**光谱 → sRGB (显示):**
```
pixelColor += Σ_b (p.spectral[b] / 40.0) × BUCKET_CMF[b]
```
每桶40nm宽带除以40得光谱密度, 乘预积分CMF得线性sRGB贡献.

**RGB → 光谱 (albedo/吸收, `albedoLegendreExpand`):**
```
反射率模型: R(λ) = dot(alb, CMF(λ)) / sum(CMF(λ))
亮度归一化: GL求积均值 = dot(alb, 1/3)
3点GL求积(±0.7746,0) → ln(R(x_k)) → Legendre拟合(c₀,c₁,c₂)
```
应用: `applyAbsorption()` — 光源Legendre状态 + 反射Legendre系数, 逐GL点相加.

**RGB → 光谱 (金属Fresnel, `reflectanceLegendreExpand`):**
```
r₀/r₁/r₂: albedo RGB → Legendre展开 (全局光谱范围)
每GL点评估: R(λ_k) = evalLegendre(r₀,r₁,r₂, (λ_k-λ_c)/λ_h)
→ ln(R_k) → Legendre拟合(c₀,c₁,c₂)
```

**RGB → 光谱 (光源/光子, `estimateSpectrum`):**
```
S(λ) = lin_R·BASIS_R(λ) + lin_G·BASIS_G(λ) + lin_B·BASIS_B(λ)
```
Wiener基函数线性组合. `storePhoton()`: 每桶中心波长评估S(λ_b), × Legendre衰减D(x_b) × 桶宽度.

**Gauss-Legendre求积 (天空/光源积分):**
7点GL求积用于宽带颜色积分: `Σ w_k × S_k × D_k × CMF_k × halfWidth`.
3点GL求积用于Legendre投影/吸收操作.

## 七、后处理

### 7.1 跨帧累积
`CrossFrameAccum`: 每像素 `newAcc = oldAcc + pixelColor`, `newCnt = oldCnt + spp`.
显示: `linearToSRGB(newAcc/max(newCnt,1) × 2.5)`. 相机移动>0.001阈值时重置.

### 7.2 降噪
2-scale边缘保持滤波器(3×3核):
- scale1(step=1, 80%权重) + scale2(step=2, 20%权重)
- 边缘停止权重: 法线差(Gaussian σ=0.14), 深度差(σ=0.32), 亮度差(σ=0.14)
- 天空像素(normal≈0)直通不过滤
